# REACH Summer Build

This is a Google project. We build it with Google products, starting with the Antigravity IDE.

## Step 1: Install the Antigravity IDE

Download and install the **[Google Antigravity IDE](https://antigravity.google/)**. Get the Antigravity *IDE*, the coding environment, not anything else by that name.

Then send me the **Google email** you want to use for this project.

## Step 2: I get you connected

Once I have your Google email, I will:

- add you to our Google Cloud project and credits,
- turn on **Gemini** inside your Antigravity IDE,
- give you the other Google tools that plug into Antigravity through our credits.

Everyone gets Gemini. That is your default assistant for the summer.

## Step 3: Add your own agent (optional)

Gemini is your default. If you also have a **Claude** or **ChatGPT** subscription and want an agent you already know:

1. In the Antigravity IDE, open the **Extensions** panel.
2. Search for your assistant and install it (for example, **Claude Code** for Claude, or the **Codex / ChatGPT** extension for ChatGPT).
3. Run it and **sign in with your existing subscription** when the extension prompts you to authenticate.

No subscription? No problem. Gemini has you covered.

## Step 4: Get the repo and your tasks

Open `morehouse-supercomputing/reach-website` in Antigravity, or:

```
git clone https://github.com/morehouse-supercomputing/reach-website.git
```

Your tasks are GitHub Issues, labeled with your role and grouped by milestone (M0 through M4). The milestones line up with the pay schedule.

## How we work

Nobody pushes to `main` but me, so the live site never breaks.

1. Branch off `main`, named for your work, like `data-viz/heatmap`.
2. Commit, push your branch, open a Pull Request into `main` with me as reviewer.
3. Your PR gets a live preview link. Check it.
4. I review and merge.

## Weekly check-in

Every Monday, a short weekly issue. Drop what you shipped and link the PR or demo. That clears your milestone for payment.

## One hard rule

This repo is public. Nothing sensitive goes in it: no benchmark data or unpublished results, no credentials, keys, or `.env` files. Those live in private Google Cloud Storage.

## Stuck?

We meet three times a week. Flag it, label the issue `blocked`, and bring it.
